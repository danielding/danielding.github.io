hexo中，如何使前端静态demo代码不被渲染

问题

开发中，经常写点静态代码demo用于演示，这种demo通常就1个html/css/js，直接在点开就能运行，不需要静态服务器啥的。
比如，以前写的一个魔方小程序，cube文件夹底下就4个文件：index.html cube.css cube.js zepto.js
可今天把这个cube文件夹放到source目录底下，hexo g之后，却变成了下图这个样子：被hexo的模板包裹了一层，而且脚本无法运行！！

真是坑爹啊o(╯□╰)o，看了下，原因是hexo对index.html做了一次渲染，导致zepto.js加载有问题。
那怎么使hexo不处理source目录下某个子目录的所有文件，仅仅是将其copy到public目录中对应目录呢？

解决方法

网上找了一圈，最终在hexo github 的issue list里找到了答案，其实很简单，hexo本身提供了这个能力。
